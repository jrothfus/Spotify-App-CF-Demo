How do the .env variables get put into the app and do they end up on the client side? (don't change any code)
Tuesday 9:51 AM


The .env variables are loaded only by server.mjs when the Node server starts.

In this app, server.mjs calls loadDotEnv(join(__dirname, ".env")), which reads the local .env file and copies values like SPOTIFY_CLIENT_ID and SPOTIFY_CLIENT_SECRET into process.env.

They do not get bundled into public/app.js, public/index.html, or anything served directly to the browser.

Flow is:

.env
  -> loaded by server.mjs into process.env
  -> used by server-side Spotify auth
  -> server calls Spotify API
  -> browser receives only chart/track/artist JSON
So the secret stays server-side. The browser can call routes like /api/charts, but it never receives SPOTIFY_CLIENT_SECRET unless the server explicitly sends it, which this app does not.
