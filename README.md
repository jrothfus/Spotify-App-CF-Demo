# Spotify App CF Demo

This repo is for testing different ways to create a small Spotify app. The goal is to compare the development experience across three scenarios.

## Prompt
```
I have trouble stayting up with current music and I want to be better about it so I can relate 
to the robotics students I mentor. Create a web app that I can use to look at the top spotify 
artists and songs. I want to be able to show these by year, by default show the current most 
popular but let me search by year. I want to be able to click on an artist and see their most 
popular tracks. Then if I click on them it opens up spotify to listen to that song.
```

## Scenarios

### 1. Using the Postman SDK generator

The `SDK-Gen` folder is for building the Spotify app with an SDK generated by the Postman SDK generator. In this scenario, development starts from a Spotify API definition or collection, generates a client SDK, and then uses that SDK inside the app code.

### 2. Creating an SDK with an AI

The `No SDK-Gen` folder is for building the Spotify app with an SDK created manually with help from an AI assistant. In this scenario, the SDK is still used by the app, but it is written or scaffolded by AI instead of generated by Postman tooling.

### 3. Not using an SDK

The `No SDK` folder is for building the Spotify app without any SDK. In this scenario, the app talks directly to Spotify's HTTP APIs using request code written in the app itself.

## Purpose

Each folder should contain the same basic Spotify app idea implemented with a different API integration strategy. This makes it easier to compare setup time, code quality, maintainability, and developer experience between generated SDKs, AI-created SDKs, and direct API calls.
